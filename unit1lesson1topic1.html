Complexity

Modern IT companies build software models of interrelated, increasingly automated services.
Customers expect these services to run 24 hours a day, 365 days a year.
Growing expectations have accelerated the complexity of these systems.
As software complexity increases, it becomes inevitable that these systems will surprise us by failing in unexpected and unusual ways.

Letâ€™s refer to these surprises as incidents.

Incidents are signals sent by the system to tell us where complexity exists. Example 1 illustrates some examples of IT incidents.

Service Outages

An unplanned interruption or reduction in the quality of an IT service.

Hardware Failures

The malfunction or breakdown of a computer's physical components.

Software Bugs

An error, flaw, or unintended behaviour in a software application that causes it to produce incorrect or unexpected results.

Security Threats

An event that jeopardises the confidentiality or integrity of an information system or the information it processes.

Deployment Issues

A disruption to normal IT services caused by releasing a software or system update.
